## TL;DR

- **Stack:** Flutter 3.41 / Dart 3.11 + Riverpod + go_router + dio +
  Hive + share_plus + camera + locus + geolocator. Pinned in
  `apps/mobile/pubspec.yaml`.
- **Strategy:** frontend-first. Every UI screen consumes an abstract
  `Repository`. Mock impls behind `Env.useMocks=true` (default) ship
  the entire 11-screen flow without a backend. Real impls are
  `UnimplementedError`-stubbed and ready for Phase 2.
- **Visual fidelity:** every screen built side-by-side against the
  HTML mock in `design/mocks/v1/smwhr _standalone_.html`. Reference
  screenshots in `docs/sprints/screenshots/mock-html-*.png`,
  result screenshots in `docs/sprints/screenshots/session-*.png`.
- **Tests:** 28/28 passing in 8 s (handle validator, onboarding
  controller, mock repos, shared widgets).
- **Smoke:** every session closed with a profile-mode screenshot on
  iPhone 16e simulator.

## Architecture

### Repository pattern (the keystone)

Every UI screen reads from an abstract repository. There are 5:

| Abstract | Mock impl | Real impl (Phase 2 stub) |
|---|---|---|
| `AuthRepository` | `MockAuthRepository` (Hive-persisted session, 800-1200 ms latency) | `RealAuthRepository` |
| `UsersRepository` | `MockUsersRepository` | `RealUsersRepository` |
| `EventsRepository` | `MockEventsRepository` (intent toggle + intent-count broadcast streams) | `RealEventsRepository` |
| `QuestsRepository` | `MockQuestsRepository` (timer simulation: 1s wall = 1 mock dwell minute, staggered check unlocks) | `RealQuestsRepository` |
| `BadgesRepository` | `MockBadgesRepository` | `RealBadgesRepository` |

Provider switching happens in `lib/data/providers.dart` based on
`Env.useMocks`. Real repos throw `UnimplementedError` with the exact
endpoint string until Phase 2 fills them in.

### Routing

`lib/core/router/app_router.dart` + `transitions.dart`:
- 14 routes (the 12 product screens + `/_debug`).
- `_AuthStateListenable` adapter so go_router re-runs redirects when
  `authStateProvider` changes.
- Redirect: signed-out → `/`. Splash, `/_debug`, and `/onboarding/*`
  are exempt. `Env.bootAt` set bypasses redirect (design QA mode).
- Custom transitions:
  - Splash + Reveal: fade-through 500 ms (ceremonial moments).
  - Onboarding 02→03→04: fade-through 320 ms (deeper levels).
  - Home: fade-through 420 ms.
  - Camera: slide-up modal 320 ms.
  - Default: platform default (iOS slide).

### Env

`lib/core/config/env.dart` — all from `--dart-define`:
- `USE_MOCKS` (default `true`)
- `API_BASE_URL` (default `https://api.smwhr.dev`)
- `SUPABASE_URL`, `SUPABASE_ANON_KEY`
- `BOOT_AT_SPLASH` — boots `/` instead of `/_debug` (design QA)
- `BOOT_AT=/some/path` — boots any path directly + bypasses auth
  redirect (design QA)

### Data layer (Phase 2 plumbing)

`lib/data/remote/`:
- `ApiClient` — Dio singleton, 10 s connect / 20 s receive, auth
  interceptor mounted at construction.
- `AuthInterceptor` (QueuedInterceptor) — adds `Authorization:
  Bearer <token>` from a pluggable `AuthTokenSource`; on 401
  attempts `tryRefresh()` + retry once.
- `Real{Auth,Users,Events,Quests,Badges}Repository` — full
  interfaces, every method `throw UnimplementedError('<exact
  endpoint> — Phase 2.')`. Phase 2 fills in `_api.dio.get(...)`
  calls without touching screens.

## How to run

```bash
# default (mock mode, debug menu boot)
cd apps/mobile && flutter run

# boot the splash screen for design review
flutter run --dart-define=BOOT_AT_SPLASH=true

# boot any specific screen
flutter run --dart-define=BOOT_AT=/profile

# Phase 2 (when backend is live)
flutter run \
  --dart-define=USE_MOCKS=false \
  --dart-define=API_BASE_URL=https://api.smwhr.dev \
  --dart-define=SUPABASE_URL=$SUPABASE_URL \
  --dart-define=SUPABASE_ANON_KEY=$SUPABASE_ANON_KEY
```

Tests:
```bash
cd apps/mobile && flutter test
# 28/28 in ~8s
```

## Gotchas & dev notes

- **`Badge` collision:** Flutter's Material library has its own
  `Badge` widget. Files that use the smwhr `Badge` model do
  `import 'package:flutter/material.dart' hide Badge;`.
- **`google_fonts` in widget tests:** set
  `GoogleFonts.config.allowRuntimeFetching = false` in `setUp` or
  the test runner hangs trying to fetch fonts.
- **`onboardingControllerProvider` is `autoDispose`:** tests must
  keep a `c.listen(...)` subscription open for the lifetime of the
  test. The mock auth has 800-1200 ms latency + 400 ms debounce, so
  `await Future.delayed(1500ms)` after `setHandle('...')` before
  asserting `handleStatus == HandleStatus.available`.
- **iOS deployment target is 15.0** (Podfile + project.pbxproj). Bumped
  from Flutter's default 13.0 to satisfy `locus ^2.0` and
  `firebase_messaging ^15.x`.
- **`permission_handler` is `^12.0.1`** (not `^11.3.1` as mobile
  CLAUDE.md says) because `locus ^2.0` requires it. Flagged in
  `pubspec.yaml` comment.
- **bg `bash` working directory** doesn't always persist across
  `run_in_background` calls. Always use absolute paths or explicit
  `cd /Volumes/Storage/Projects/smwhr/apps/mobile && flutter ...`.
- **Profile mode is not supported on iOS simulator.** Real-device
  perf testing waits for hardware.
- **Hive auth box** name is `mock_auth`. Cold restart rehydrates the
  signed-in user. Phase 2 will rename it to `auth_session` (same
  schema in `AuthSession.toJson()`).
